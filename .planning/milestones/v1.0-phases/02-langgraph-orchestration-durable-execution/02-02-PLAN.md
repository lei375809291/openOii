# Phase 2 / Plan 02 — LangGraph durable orchestration core

> **For agentic workers:** REQUIRED: Use `superpowers:subagent-driven-development` if subagents are available, or `superpowers:executing-plans` to implement this plan.

**Goal:** Move the generation engine onto a durable LangGraph pipeline with explicit approval gates and checkpoint-backed resume.

**Architecture:** Keep `Run.thread_id` as the single durable cursor, compile the root graph with a PostgreSQL checkpointer, and let the graph own the story → characters → storyboards → clips → merge flow. The existing manual orchestrator remains the behavioral reference, but its loop and rerun logic should be replaced by graph execution, stage nodes, and resume-safe side effects.

**Tech Stack:** LangGraph 1.x, langgraph-checkpoint-postgres, FastAPI, SQLModel, Redis-backed confirmation bridge.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API/runtime → LangGraph | Untrusted request state must not become graph state without validation. |
| Graph runtime → PostgreSQL checkpoints | Resume data must stay bound to the correct project and thread. |
| Graph nodes → provider side effects | Re-entered nodes must not duplicate external writes or media generation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04 | Spoofing | `thread_id` ownership | mitigate | Bind every resume to authenticated project ownership and the persisted `Run.thread_id`. |
| T-02-05 | Tampering | `interrupt()` resume data | mitigate | Validate resume payloads at the graph boundary and keep approval gates explicit. |
| T-02-06 | Repudiation | resumed provider calls | mitigate | Make side effects idempotent and persist stage lineage before replay can touch downstream outputs. |
</threat_model>
