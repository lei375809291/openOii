# Phase 2 / Plan 01 — Durable execution regression contract

> **For agentic workers:** REQUIRED: Use `superpowers:subagent-driven-development` if subagents are available, or `superpowers:executing-plans` to implement this plan.

**Goal:** Lock the durable-execution contract in tests before the orchestration migration lands.

**Architecture:** This plan captures the Phase 2 promises as executable regressions: durable `thread_id` ownership, checkpoint-backed resume from the last valid stage, and explicit rejection of duplicate active runs. Keep the test harness aligned with the existing SQLModel/pytest style so later production changes can satisfy the same contract without rewriting the fixtures.

**Tech Stack:** pytest, pytest-asyncio, SQLModel, FastAPI test client, LangGraph persistence contract.

Make the Phase 2 behavior concrete enough that the later LangGraph migration has a hard target to satisfy.

Purpose: prevent silent regressions while moving from manual orchestration to durable execution.
Output: backend regression tests plus any fixture support needed for durable checkpoint validation.
</objective>
