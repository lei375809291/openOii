# Final Assembly & Delivery Backend Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Keep the last known final video visible while downstream clips rerun, let creators retry merge from the final-output card with the same run/thread recovery semantics, and expose a safe product download route for the merged asset.

**Architecture:** Reuse the current canonical `project.video_url` as the only final asset pointer, but stop clearing it when reruns invalidate downstream clips. Mark the project as `superseded` while the current clips are rebuilding, then restore a non-superseded state when a new merge succeeds. Treat the existing `/feedback` run as the retry-merge entry point from the final-output card, and keep the recovery boundary on the same run/thread so retry behaves like a review-triggered rerun instead of a second delivery workflow. Add a controlled `FileResponse` download endpoint that resolves the stored local asset through the existing static-file safety helper.

**Tech Stack:** FastAPI, SQLModel, pytest, FileResponse, existing async test harness
