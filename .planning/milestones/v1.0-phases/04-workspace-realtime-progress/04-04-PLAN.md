# Phase 04 — Workspace Shell Wiring Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the projected canvas the default workspace shell so empty projects still open into the backend-authored section view.

**Architecture:** Reuse the canvas projection from Plan 02, then keep the StageView/InfiniteCanvas bridge thin and deterministic. This plan only handles the shell handoff and refresh-safe wiring; progress synchronization remains in Plan 03.

**Tech Stack:** React 19, TypeScript, tldraw, Vitest

***

## Context

Phase 4’s shell must open into the projected workspace instead of the old ideate-only fallback. The canvas remains a backend-authored projection, and this plan only wires the shell and empty-state fallbacks so the creator sees the same workspace after refresh.

Key files: `frontend/app/components/layout/StageView.tsx`, `frontend/app/components/canvas/InfiniteCanvas.tsx`, and their tests. Keep this slice isolated from websocket/progress work so the 04-03 plan stays focused.
