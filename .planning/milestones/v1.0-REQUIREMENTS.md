# v1.0 Requirements Archive

**Archived:** 2026-04-12  
**Status:** shipped with deferred warning

## v1 Requirements Outcomes

### Project Setup

- [x] **PROJ-01**: A creator can create a project by entering a story idea, title, and style direction.  
  **Outcome:** validated in Phase 01 + Phase 04 end-to-end workspace flow.

- [x] **PROJ-02**: A creator can configure the text, image, and video providers used by the project before running generation.  
  **Outcome:** accepted as a **deferred portability warning** for v1.0. Global provider configuration and precedence exist, but project-scoped provider swap is not end-to-end validated.

### Pipeline Execution

- [x] **PIPE-01**: A creator can trigger a full generation run that executes the end-to-end pipeline from story input to final merged video.  
  **Outcome:** validated in Phase 02.

- [x] **PIPE-02**: A creator can see the current stage, progress, and status changes of a generation run in real time.  
  **Outcome:** validated in Phase 04.

- [x] **PIPE-03**: A creator can receive a final merged video artifact when all required clips complete successfully.  
  **Outcome:** validated in Phase 06.

### Character and Story Consistency

- [x] **CHAR-01**: A creator can upload or define character reference inputs that are reused by downstream generation stages.  
  **Outcome:** validated in Phase 03.

- [x] **CHAR-02**: The system can preserve character identity consistently across storyboard and video outputs for the same project.  
  **Outcome:** validated in Phase 03.

- [x] **SHOT-01**: A creator can review storyboard outputs per shot before or during downstream video generation.  
  **Outcome:** validated in Phase 03.

### Workspace and Visualization

- [x] **WORK-01**: A creator can use an infinite canvas workspace to view the project’s script, characters, storyboards, clips, and final output as related artifacts.  
  **Outcome:** validated in Phase 04.

- [x] **WORK-02**: The workspace can show artifact status clearly, including draft, generating, complete, failed, and superseded states.  
  **Outcome:** validated in Phase 04.

### Review and Creative Control

- [x] **REVI-01**: A creator can selectively regenerate a single character, storyboard shot, or video clip without restarting the full project run.  
  **Outcome:** validated in Phase 05.

- [x] **REVI-02**: A creator can edit the prompt or generation instructions for a specific shot before rerunning it.  
  **Outcome:** validated in Phase 05.

### Reliability and Recovery

- [x] **REL-01**: A creator can resume a run after interruption or failure from the last valid stage instead of restarting the entire pipeline.  
  **Outcome:** validated in Phase 02.

### Delivery

- [x] **DELIV-01**: A creator can preview the final merged video inside the product.  
  **Outcome:** validated in Phase 06.

- [x] **DELIV-02**: A creator can download the final merged video once generation is complete.  
  **Outcome:** validated in Phase 06.

## v1.x Deferred

- PROJ-03, WORK-03, REVI-03, REL-02, CTRL-01, CTRL-02, MEDIA-01 remain deferred beyond v1.0.
