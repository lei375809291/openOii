# Phase 04 — Workspace Status Contract Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create a shared, pure projection layer that turns backend project/run/artifact state into canonical workspace sections and creator-facing status labels.

**Architecture:** Keep the projection backend-authored and side-effect free. This helper becomes the single source of truth for canvas slot states and creator-friendly run labels, so later UI tasks can render the same status vocabulary without duplicating mapping logic.

**Tech Stack:** TypeScript, Vitest, React 19

---

## Context

Phase 4 is a backend-authored projection problem, not a free-form canvas rewrite. The user locked a stage-section-first layout, canonical placeholders for missing output, and creator-friendly progress labels; research confirmed the safest path is one deterministic projection helper that all Phase 4 surfaces consume.

Relevant existing files: `frontend/app/hooks/useCanvasLayout.ts`, `frontend/app/components/canvas/InfiniteCanvas.tsx`, `frontend/app/components/layout/StageView.tsx`, `frontend/app/hooks/useWebSocket.ts`, and `frontend/app/pages/ProjectPage.tsx`. This plan starts with the shared status contract so the later UI tasks can stay small and consistent.
