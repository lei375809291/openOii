# Phase 04 — Canvas Workspace Rendering Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Render the creator workspace as canonical section cards with placeholders and status badges through the shared canvas projection, while keeping the section-first layout deterministic.

**Architecture:** Consume the shared projection helper from Plan 01, then thread the resulting section/status data through `useCanvasLayout` and the tldraw shape props. The canvas should surface state without becoming the primary progress console; shell wiring lives in a separate plan to keep this slice small.

**Tech Stack:** React 19, TypeScript, tldraw, Vitest, Tailwind CSS

***

## Context

The phase boundary requires a stage-section-first canvas with canonical script, characters, storyboards, clips, and final output sections. Missing content must still show as placeholders, and the progress shell—not the canvas—remains the main explanation of what the run is doing.

The key files are `frontend/app/hooks/useCanvasLayout.ts`, `frontend/app/components/canvas/shapes/*`, and `frontend/app/components/canvas/VideoSectionShape.tsx`. This plan makes the canvas render the projection from Plan 01 instead of hiding empty sections or relying on user-authored canvas state.
