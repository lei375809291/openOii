# Phase 05 — Review & Creative Control Frontend Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan. Steps use checkbox syntax for tracking.

**Goal:** Make selective rerun controls and lineage labels visible and trustworthy in the creator workspace.

**Architecture:** Keep the workspace shell and tldraw canvas as the primary surface. Extend the existing modal/action/store flow so the creator can edit shot inputs before rerunning and can see current vs superseded states without a version browser.

This plan intentionally stops at shared control-flow and status-projection work; per-shape visual polish is excluded so the slice stays inside the execution sweet spot.

**Tech Stack:** React 19, TypeScript, Zustand, TanStack Query, tldraw, Vitest, Testing Library

---

## Context

The backend contract already exposes approval-aware characters and shots, resource-scoped reruns, and workspace status labels such as `current`, `superseded`, and `waiting-for-review`. This plan only needs to surface that contract cleanly in the existing canvas shell and edit modal flow.

The key user promise is simple: edit one thing, rerun only what depends on it, and understand what became current, what became superseded, and why.
