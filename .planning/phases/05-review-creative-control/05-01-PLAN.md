# Phase 05 — Review & Creative Control Backend Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan. Steps use checkbox syntax for tracking.

**Goal:** Make selective reruns safe, scoped, and acceptance-driven on the backend.

**Architecture:** Reuse the existing resource-scoped regenerate routes, approval snapshots, and review routing. Add explicit candidate/current promotion plus one-hop invalidation helpers so the server, not the client, owns lineage truth.

**Tech Stack:** FastAPI, SQLModel, pytest, pytest-asyncio, existing LangGraph/agent routing

---

## Context

Phase 5 is state choreography, not new generation logic. The locked rule is: a targeted rerun only affects the selected artifact and its direct downstream dependents, while acceptance promotes the new result into current state and supersedes the previous one.

Relevant code paths already exist in `backend/app/api/v1/routes/characters.py`, `backend/app/api/v1/routes/shots.py`, `backend/app/agents/review.py`, `backend/app/models/project.py`, `backend/app/models/artifact.py`, and `backend/app/schemas/project.py`. The missing work is to make that contract explicit, tested, and server-authoritative.
