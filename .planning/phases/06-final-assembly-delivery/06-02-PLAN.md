# Final Assembly & Delivery Frontend Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Surface the assembled final video in the existing workspace, with clear current/stale labeling, a reliable download action, and a retry-merge button that reuses the same review/run/thread recovery path.

**Architecture:** Keep the final-output section as the canonical delivery surface. The workspace projection should continue to source from `project.video_url`, but the UI should expose provenance/status so a stale final is never mistaken for current. Add a retry action to the same final-output surface, wired to the existing `/feedback` API and current run/thread context so retry behaves like a review-triggered rerun instead of a separate delivery flow. Both the canvas card and the overview panel should point to the same controlled download behavior and share the same hydration path through websocket-driven store updates.

**Tech Stack:** React, TypeScript, tldraw, Vitest, Playwright, existing editor store and websocket hydration
